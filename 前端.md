## agent 如何具体调用的react hook, 然后如何渲染的

### 📋 架构概览

```
用户输入 → React 前端 → CopilotKit Runtime → LangGraph Agent → 返回组件调用 → 渲染 React 组件
```

---

## 第一步：前端注册组件（`useComponent`）

### 1.1 在应用启动时注册所有组件

在 [page.tsx](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/app/page.tsx#L15) 中：

```typescript
export default function HomePage() {
  useGenerativeUIExamples(); // ← 注册所有生成式 UI 组件
  // ...
}
```

### 1.2 `useGenerativeUIExamples` 注册组件

在 [use-generative-ui-examples.tsx](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/hooks/use-generative-ui-examples.tsx#L39) 中：

```typescript
export const useGenerativeUIExamples = () => {
  // 注册饼图组件
  useComponent({
    name: "pieChart",
    description: "将数据显示为饼图。",
    parameters: PieChartProps, // Zod schema 定义参数
    render: PieChart, // React 组件
  });

  // 注册 Widget 渲染器（最灵活的组件）
  useComponent({
    name: "widgetRenderer",
    description: "在沙盒化 iframe 中渲染交互式 HTML/SVG 可视化。",
    parameters: WidgetRendererProps,
    render: WidgetRenderer,
  });
};
```

### 1.3 `useComponent` 的工作原理

`useComponent` 做了三件事：

1. **注册组件**：将组件名称、描述、参数 schema 和渲染函数注册到 CopilotKit 的内部注册表
2. **发送给 Agent**：通过 WebSocket 或 HTTP 将组件信息发送给 Agent，让 Agent 知道有哪些可用的组件
3. **等待调用**：当 Agent 返回组件调用时，CopilotKit 会自动渲染对应的 React 组件

## 第三步：CopilotKit 处理响应

### 3.1 API 路由处理响应

在 [route.ts](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/app/api/copilotkit/route.ts#L47) 中：

```typescript
export const POST = async (req: NextRequest) => {
  const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
    endpoint: "/api/copilotkit",
    serviceAdapter: new ExperimentalEmptyAdapter(),
    runtime: new CopilotRuntime({
      agents: { default: defaultAgent }, // ← LangGraph Agent
      a2ui: { injectA2UITool: true },
    }),
  });

  return handleRequest(req); // ← 处理请求并返回响应
};
```

### 3.2 CopilotKit Runtime 解析响应

CopilotKit Runtime 会：

1. **解析 Agent 响应**：识别 `tool_calls` 中的组件调用
2. **匹配组件**：根据 `name` 查找注册的组件（如 `widgetRenderer`）
3. **验证参数**：使用 Zod schema 验证参数是否符合定义
4. **返回给前端**：将组件调用信息返回给前端

## 第四步：前端渲染组件

### 4.1 `useAgent` 接收响应

在 [page.tsx](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/app/page.tsx#L17) 中：

`useAgent` hook 会：
- 监听 Agent 的响应
- 解析组件调用
- 自动渲染对应的 React 组件

### 4.2 `CopilotChat` 渲染消息

在 [page.tsx](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/app/page.tsx#L95) 中：

`CopilotChat` 组件会：
- 显示所有消息（包括文本消息和组件调用）
- 对于组件调用，自动调用对应的 `render` 函数

### 4.3 渲染 `widgetRenderer` 组件

当 Agent 调用 `widgetRenderer` 时，CopilotKit 会：

1. **查找注册的组件**：找到 `useComponent({ name: "widgetRenderer", ... })`
2. **调用 render 函数**：`render: WidgetRenderer`
3. **传递参数**：将 Agent 返回的参数传递给组件

在 [widget-renderer.tsx](file:///Users/apple/zhihe/OpenGenerativeUI/apps/app/src/components/generative-ui/widget-renderer.tsx#L1) 中：

```typescript
export function WidgetRenderer({ title, description, html }: WidgetRendererProps) {
  // 1. 创建沙盒化的 iframe
  // 2. 注入主题 CSS、SVG 类、表单样式
  // 3. 注入桥接 JS（sendPrompt, openLink）
  // 4. 使用 postMessage 将 HTML 流式传输到 iframe
  // 5. 使用 Idiomorph 进行 DOM diff/patch
  // 6. 自动调整 iframe 高度
  
  return (
    <iframe
      srcDoc={assembleShell(initialHtml)}
      className="w-full border-0"
      style={{ minHeight: '200px' }}
    />
  );
```

## 第五步：Widget 渲染器的特殊机制

### 5.1 iframe 沙盒环境

`WidgetRenderer` 创建一个沙盒化的 iframe，包含：

**CSS 层**：
- 主题变量（亮/暗模式）
- SVG 预构建类（`.c-purple`, `.c-teal` 等）
- 表单样式（按钮、输入框等）

**JS 层**：
- 桥接函数（`sendPrompt`, `openLink`）
- 自动调整高度（`ResizeObserver`）
- DOM diff/patch（Idiomorph）

**Import Map**：
```html
<script type="importmap">
{
  "imports": {
    "three": "https://esm.sh/three",
    "gsap": "https://esm.sh/gsap",
    "d3": "https://esm.sh/d3",
    "chart.js": "https://esm.sh/chart.js"
  }
}
</script>
```

### 5.2 postMessage 通信

父窗口 → iframe：
```javascript
window.iframe.contentWindow.postMessage({
  type: 'update-content',
  html: agentGeneratedHtml
}, '*');
```

iframe → 父窗口：
```javascript
window.parent.postMessage({
  type: 'widget-resize',
  height: contentHeight
}, '*');

window.parent.postMessage({
  type: 'send-prompt',
  text: userPrompt
}, '*');
```

### 5.3 DOM Diff/Patch

使用 Idiomorph 进行高效的 DOM 更新：

```javascript
Idiomorph.morph(content, tmp, {
  morphStyle: 'innerHTML',
  callbacks: {
    beforeNodeAdded: function(node) {
      // 为新元素添加入场动画
      if (node.nodeType === 1) {
        node.classList.add('morph-enter');
      }
    }
  }
});
```

## 关键技术点

### 1. 组件注册（`useComponent`）
- 在应用启动时注册所有可用组件
- 将组件信息发送给 Agent
- Agent 根据描述决定何时调用哪个组件

### 2. 参数验证（Zod Schema）
```typescript
const WidgetRendererProps = z.object({
  title: z.string(),
  description: z.string(),
  html: z.string(),
});
```
- 确保 Agent 返回的参数符合预期
- 提供类型安全

### 3. 沙盒化 iframe
- 隔离 Agent 生成的代码
- 防止 XSS 攻击
- 提供安全的执行环境

### 4. postMessage 通信
- 父窗口和 iframe 之间的双向通信
- 支持流式更新
- 自动调整高度

### 5. DOM Diff/Patch
- 使用 Idiomorph 进行高效的 DOM 更新
- 保留现有节点，避免闪烁
- 只更新变化的部分

---

## 总结

Agent 调用 React Hook 的核心机制：

1. **前端注册**：使用 `useComponent` 注册组件，CopilotKit 将组件信息发送给 Agent
2. **Agent 决策**：Agent 根据用户请求和系统提示决定调用哪个组件
3. **返回调用**：Agent 返回组件名称和参数
4. **自动渲染**：CopilotKit 自动查找并渲染对应的 React 组件
5. **特殊处理**：`widgetRenderer` 使用 iframe 沙盒环境，通过 postMessage 通信

这种机制让 Agent 能够动态生成和渲染复杂的 UI 组件，而无需硬编码任何前端逻辑。
